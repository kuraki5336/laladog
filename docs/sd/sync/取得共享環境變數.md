## API 總覽

取得指定團隊的共享環境變數。需要 JWT 認證，且當前使用者必須是該團隊的成員（任意角色皆可）。每個團隊最多一筆環境變數記錄，`data` 欄位為所有環境與變數的 JSON 字串。

---

## Request 規格

### HTTP Method & Path

```
GET /sync/{team_id}/environments
```

### Headers

| 名稱          | 必填 | 說明                       |
| ------------- | ---- | -------------------------- |
| Authorization | 是   | `Bearer {JWT token}`       |

### Path Parameters

| 參數    | 型別   | 必填 | 說明              |
| ------- | ------ | ---- | ----------------- |
| team_id | string | 是   | 團隊 ID（UUID）   |

### Query Parameters

無。

### Request Body

無。

---

## Response 規格

### 成功回應 — 200 OK（有資料）

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "team_id": "t1a2b3c4-d5e6-7890-abcd-ef1234567890",
  "data": "[{\"id\":\"uuid\",\"name\":\"Development\",\"variables\":[{\"id\":\"uuid\",\"key\":\"baseUrl\",\"value\":\"http://localhost:8000\",\"enabled\":true}]}]",
  "updated_by": "u1a2b3c4-d5e6-7890-abcd-ef1234567890",
  "updated_at": "2026-03-14T08:30:00+00:00"
}
```

### 成功回應 — 200 OK（無資料）

```json
null
```

當該團隊尚未建立任何環境變數時回傳 `null`。

### 回應欄位說明

| 欄位       | 型別   | 說明                                              |
| ---------- | ------ | ------------------------------------------------- |
| id         | string | SharedEnvironment 的 UUID                         |
| team_id    | string | 所屬團隊 ID                                       |
| data       | string | 環境變數 JSON 字串（陣列格式，包含所有環境與變數） |
| updated_by | string | 最後更新者的使用者 ID                             |
| updated_at | string | 最後更新時間（ISO 8601 格式）                     |

### `data` 欄位 JSON 結構

```json
[
  {
    "id": "環境 UUID",
    "name": "環境名稱",
    "variables": [
      {
        "id": "變數 UUID",
        "key": "變數名稱",
        "value": "變數值",
        "enabled": true
      }
    ]
  }
]
```

## 實作說明

### 處理流程

1. 從 JWT token 中取得當前使用者 ID（`current_user["sub"]`）
2. 查詢 `team_members` 資料表，驗證該使用者是否為指定 `team_id` 的成員
3. 若不是成員，回傳 403 錯誤
4. 查詢 `shared_environments` 資料表，取得該 `team_id` 的記錄
5. 若無記錄回傳 `null`，否則轉換為 `EnvironmentResponse` 格式回傳

### 關鍵程式碼位置

- 路由函式：`backend/routers/sync.py` → `get_shared_environments()`
- 資料模型：`backend/models/team.py` → `SharedEnvironment`, `TeamMember`

### 權限驗證邏輯

```python
result = await db.execute(
    select(TeamMember).where(
        TeamMember.team_id == team_id,
        TeamMember.user_id == user_id
    )
)
if not result.scalar_one_or_none():
    raise HTTPException(status_code=403, detail="Not a team member")
```

- 僅檢查是否為團隊成員，不限制角色（admin / editor / viewer 皆可）

---

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息               | 觸發條件                           |
| ----------- | ---------------------- | ---------------------------------- |
| 401         | Unauthorized           | 未提供或無效的 JWT token           |
| 403         | Not a team member      | 當前使用者不是指定團隊的成員       |

## 測試案例

### 正常情境

| # | 測試案例                              | 預期結果                                    |
| - | ------------------------------------- | ------------------------------------------- |
| 1 | admin 角色取得團隊的環境變數           | 200，回傳 EnvironmentResponse 物件           |

| # | 測試案例                              | 預期結果                               |
| - | ------------------------------------- | -------------------------------------- |
| 1 | 未攜帶 JWT token                      | 401 Unauthorized                       |
| 2 | JWT token 過期或無效                  | 401 Unauthorized                       |
| 3 | 使用者不是該團隊成員                  | 403 Not a team member                  |
