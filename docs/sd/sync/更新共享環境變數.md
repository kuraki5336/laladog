## API 總覽

更新指定團隊的共享環境變數（整包覆蓋）。需要 JWT 認證，且當前使用者必須是該團隊的 editor 或 admin。若該團隊尚無環境變數記錄則自動建立（upsert 語意）。

---

## Request 規格

### HTTP Method & Path

```
PUT /sync/{team_id}/environments
```

### Headers

| 名稱          | 必填 | 說明                       |
| ------------- | ---- | -------------------------- |
| Authorization | 是   | `Bearer {JWT token}`       |
| Content-Type  | 是   | `application/json`         |

### Path Parameters

| 參數    | 型別   | 必填 | 說明              |
| ------- | ------ | ---- | ----------------- |
| team_id | string | 是   | 團隊 ID（UUID）   |

### Query Parameters

無。

### Request Body

```json
{
  "data": "[{\"id\":\"uuid\",\"name\":\"Development\",\"variables\":[{\"id\":\"uuid\",\"key\":\"baseUrl\",\"value\":\"http://localhost:8000\",\"enabled\":true}]}]"
}
```

| 欄位 | 型別   | 必填 | 說明                                                    |
| ---- | ------ | ---- | ------------------------------------------------------- |
| data | string | 是   | 環境變數 JSON 字串（陣列格式，包含所有環境與其變數）     |

### `data` 欄位 JSON 結構

```json
[
  {
    "id": "環境 UUID",
    "name": "環境名稱",
    "variables": [
      {
        "id": "變數 UUID",
        "key": "變數名稱",
        "value": "變數值",
        "enabled": true
      }
    ]
  }
]
```

## Response 規格

### 成功回應 — 200 OK

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "team_id": "t1a2b3c4-d5e6-7890-abcd-ef1234567890",
  "data": "[{\"id\":\"uuid\",\"name\":\"Development\",\"variables\":[...]}]",
  "updated_by": "u1a2b3c4-d5e6-7890-abcd-ef1234567890",
  "updated_at": "2026-03-14T08:30:00+00:00"
}
```

### 回應欄位說明

| 欄位       | 型別   | 說明                                              |
| ---------- | ------ | ------------------------------------------------- |
| id         | string | SharedEnvironment 的 UUID                         |
| team_id    | string | 所屬團隊 ID                                       |
| data       | string | 環境變數 JSON 字串（與 Request Body 中的 data 相同）|
| updated_by | string | 當前操作者的使用者 ID                             |
| updated_at | string | 更新時間（ISO 8601 格式）                         |

## 實作說明

### 處理流程

1. 從 JWT token 中取得當前使用者 ID（`current_user["sub"]`）
2. 查詢 `team_members` 資料表，驗證該使用者是否為指定 `team_id` 的 editor 或 admin
3. 若權限不足，回傳 403 錯誤
4. 查詢 `shared_environments` 資料表，取得該 `team_id` 的記錄
5. 若已有記錄 → 更新 `data`、`updated_by`、`updated_at` 欄位
6. 若無記錄 → 建立新的 `SharedEnvironment` 記錄（upsert）
7. commit 並回傳更新後的 `EnvironmentResponse`

### 關鍵程式碼位置

- 路由函式：`backend/routers/sync.py` → `update_shared_environments()`
- 資料模型：`backend/models/team.py` → `SharedEnvironment`, `TeamMember`

### 權限驗證邏輯

```python
result = await db.execute(
    select(TeamMember).where(
        TeamMember.team_id == team_id,
        TeamMember.user_id == user_id,
        TeamMember.role.in_(["admin", "editor"]),
    )
)
if not result.scalar_one_or_none():
    raise HTTPException(status_code=403, detail="Requires editor or admin role")
```

- 僅 admin 和 editor 可更新，viewer 無寫入權限

### Upsert 邏輯

```python
# 查找現有記錄
result = await db.execute(
    select(SharedEnvironment).where(SharedEnvironment.team_id == team_id)
)
env = result.scalar_one_or_none()

if env:
    # 更新
    env.data = body.data
    env.updated_by = user_id
    env.updated_at = datetime.now(timezone.utc)
else:
    # 新建
    env = SharedEnvironment(team_id=team_id, data=body.data, updated_by=user_id)
    db.add(env)
```

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                      | 觸發條件                                     |
| ----------- | ----------------------------- | -------------------------------------------- |
| 401         | Unauthorized                  | 未提供或無效的 JWT token                     |
| 403         | Requires editor or admin role | 當前使用者不是團隊成員，或角色為 viewer       |
| 422         | Validation Error              | Request Body 格式不正確                       |

## 測試案例

### 正常情境

| # | 測試案例                                    | 預期結果                                    |
| - | ------------------------------------------- | ------------------------------------------- |
| 1 | admin 首次上傳環境變數（尚無記錄）           | 200，建立新記錄並回傳                        |
| 2 | editor 更新已有的環境變數                    | 200，覆蓋舊記錄並回傳                        |
| 3 | admin 上傳空陣列 `[]` 清空所有環境           | 200，data 為 `[]`                            |
| 4 | 上傳多個環境，各含多個變數                   | 200，完整儲存所有環境與變數                   |

### 異常情境

| # | 測試案例                                    | 預期結果                                    |
| - | ------------------------------------------- | ------------------------------------------- |
| 1 | 未攜帶 JWT token                            | 401 Unauthorized                            |
| 2 | JWT token 過期或無效                        | 401 Unauthorized                            |
| 3 | viewer 角色嘗試更新                          | 403 Requires editor or admin role           |
| 4 | 非團隊成員嘗試更新                           | 403 Requires editor or admin role           |
| 5 | Request Body 缺少 data 欄位                 | 422 Validation Error                        |
