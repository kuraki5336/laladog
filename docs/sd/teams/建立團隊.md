# 建立團隊 API

| 項目     | 說明                            |
| -------- | ------------------------------- |
| API 路徑 | `POST /teams/`                  |
| 版本     | 2.0                            |

| 狀態     | 已實作                         |

## 相關資料表

| 資料表       | 用途                               |
| ------------ | ---------------------------------- |
| teams        | 新增團隊記錄                       |
| team_members | 自動新增建立者為 admin 角色的成員  |

## API 總覽

建立新團隊，建立者自動成為該團隊的管理員（admin）。建立流程會在同一個交易中完成兩個操作：新增 `teams` 記錄，以及新增一筆 `team_members` 記錄（角色為 admin）。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |
| Content-Type  | 是   | `application/json`            |

### Path Parameters

無。

### Query Parameters

無。

### Request Body

| 欄位 | 型別   | 必填 | 說明     | 限制         |
| ---- | ------ | ---- | -------- | ------------ |
| name | string | 是   | 團隊名稱 | 不可為空字串 |

### Request 範例

```http
POST /teams/ HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
  "name": "研發一組"
}
```

## Response 規格

### 成功回應（200 OK）

| 欄位     | 型別   | 說明                       |
| -------- | ------ | -------------------------- |
| id       | string | 新建團隊的 UUID            |
| name     | string | 團隊名稱                   |
| owner_id | string | 團隊建立者的 user ID       |
| role     | string | 固定為 `"admin"`           |
| members  | array  | 成員列表（預設為空陣列）   |

### Response 範例

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "研發一組",
  "owner_id": "u001-user-uuid",
  "role": "admin",
  "members": []
}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. 建立 `Team` 物件，設定 `name` 及 `owner_id`
3. 寫入資料庫並執行 `flush()`，取得自動產生的 `team.id`
4. 建立 `TeamMember` 物件，設定 `team_id`、`user_id`、`role="admin"`
5. 寫入資料庫並執行 `commit()`
6. 組裝 `TeamResponse` 回傳

### 交易處理

- 步驟 2~5 在同一個資料庫交易中完成
- 使用 `flush()` 確保 `team.id` 已產生後，才建立 `TeamMember`
- 最後統一 `commit()`，確保 Team 與 TeamMember 同時寫入

### 程式碼位置

- Router：`backend/routers/teams.py` → `create_team()`
- Model：`backend/models/team.py` → `Team`, `TeamMember`
- Schema：`backend/routers/teams.py` → `CreateTeamRequest`, `TeamResponse`

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                    | 觸發條件                      |
| ----------- | --------------------------- | ----------------------------- |
| 401         | Invalid or expired token    | JWT Token 無效或過期          |
| 403         | Not authenticated           | 未提供 Authorization header   |
| 422         | Validation Error            | Request body 格式錯誤或缺少必填欄位 |

## 測試案例

### 測試案例 1：成功建立團隊

**前置條件**：使用者已登入

**步驟**：
1. 以有效 JWT Token 發送 `POST /teams/`，body 為 `{"name": "測試團隊"}`

**預期結果**：
- HTTP 200
- 回傳 JSON 物件包含新團隊資訊
- `role` 欄位為 `"admin"`
- `owner_id` 等於當前使用者 ID
- 資料庫中 `teams` 表新增一筆記錄
- 資料庫中 `team_members` 表新增一筆 role="admin" 的記錄

### 測試案例 2：缺少 name 欄位

**前置條件**：使用者已登入

**步驟**：
1. 以有效 JWT Token 發送 `POST /teams/`，body 為 `{}`

**預期結果**：
- HTTP 422
- 回傳 Validation Error

### 測試案例 3：未認證請求

**前置條件**：無

**步驟**：
1. 不帶 Authorization header 發送 `POST /teams/`

**預期結果**：
- HTTP 403
- 回傳錯誤訊息
