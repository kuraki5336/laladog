# 移除成員 / 離開團隊 API

## 版本資訊

| 項目     | 說明                                                    |
| -------- | ------------------------------------------------------- |
| API 路徑 | `DELETE /teams/{team_id}/members/{member_user_id}`      |
| 版本     | 2.0                                                    |
| 最後更新 | 2026-03-04                                             |
| 狀態     | 已實作                                                 |

## 相關資料表

| 資料表              | 用途                                     |
| ------------------- | ---------------------------------------- |
| team_members        | 驗證操作者權限、查找並刪除成員記錄       |
| pending_invitations | 取消未註冊使用者的待確認邀請             |

## API 總覽

此 API 有三種使用情境：

1. **離開團隊（Self-removal）**：`member_user_id` 等於當前使用者 ID，任何角色皆可操作。Admin 離開前需確保團隊中至少還有一位其他 Admin。
2. **移除他人**：`member_user_id` 為其他成員的 ID，僅 Admin 可操作。
3. **取消 pending 邀請**：若 `member_user_id` 對應到 `pending_invitations` 的 ID（而非 `team_members`），僅 Admin 可操作。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |

### Path Parameters

| 名稱           | 型別   | 必填 | 說明                                              |
| -------------- | ------ | ---- | ------------------------------------------------- |
| team_id        | string | 是   | 團隊 UUID                                        |
| member_user_id | string | 是   | 要移除的成員 user ID 或 pending invitation ID（離開時傳自己的 user ID） |

### Query Parameters

無。

### Request Body

無。

### Request 範例（移除他人）

```http
DELETE /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890/members/u002-user-uuid HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Request 範例（離開團隊）

```http
DELETE /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890/members/自己的user-uuid HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## Response 規格

### 成功回應（200 OK）

| 欄位    | 型別   | 說明               |
| ------- | ------ | ------------------ |
| message | string | 操作結果訊息       |

### Response 範例

```json
// 離開團隊
{"message": "Left team"}

// 移除成員
{"message": "Member removed"}

// 取消 pending 邀請
{"message": "Invitation cancelled"}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. 判斷 `is_self = member_user_id == user_id`
3. **若 is_self（離開團隊）**：
   a. 查詢 `team_members` 表確認自己是團隊成員
   b. 若不是成員 → 回傳 404
   c. 若自己角色為 `admin`，查詢是否有其他 `admin` 存在
   d. 若無其他 admin → 回傳 400（「Cannot leave: you are the only admin」）
   e. 刪除自己的 `TeamMember` 記錄
   f. 回傳 `{"message": "Left team"}`
4. **若非 is_self（移除他人）**：
   a. 查詢 `team_members` 表確認操作者角色為 `admin`
   b. 若非 admin → 回傳 403
   c. 先嘗試在 `team_members` 中找 `member_user_id`
   d. 若找到 → 刪除該成員記錄，回傳 `{"message": "Member removed"}`
   e. 若找不到 → 嘗試在 `pending_invitations` 中以 ID 查找
   f. 若找到 → 刪除該邀請記錄，回傳 `{"message": "Invitation cancelled"}`
   g. 皆找不到 → 回傳 404

### 權限控管

| 情境 | 權限要求 |
| ---- | -------- |
| 離開團隊（is_self） | 任何角色皆可；admin 需有其他 admin |
| 移除他人 | 僅 admin |
| 取消 pending 邀請 | 僅 admin |

### 程式碼位置

- Router：`backend/routers/teams.py` → `remove_member()`
- Model：`backend/models/team.py` → `TeamMember`, `PendingInvitation`

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                              | 觸發條件                              |
| ----------- | ------------------------------------- | ------------------------------------- |
| 400         | Cannot leave: you are the only admin... | Admin 離開但無其他 Admin              |
| 401         | Invalid or expired token              | JWT Token 無效或過期                  |
| 403         | Not authenticated                     | 未提供 Authorization header           |
| 403         | Only admin can remove members         | 非 admin 嘗試移除他人                 |
| 404         | Not a team member                     | 離開時自己不是團隊成員                |
| 404         | Member not found                      | 指定的 member_user_id 不存在          |

## 測試案例

### 測試案例 1：admin 成功移除成員

**前置條件**：使用者為團隊 admin，目標成員存在於該團隊中

**步驟**：
1. 以 admin 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{member_user_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Member removed"}`

### 測試案例 2：非 admin 嘗試移除成員

**前置條件**：使用者為團隊 editor 或 viewer

**步驟**：
1. 以 editor/viewer 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{other_user_id}`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only admin can remove members"}`

### 測試案例 3：editor 離開團隊

**前置條件**：使用者為團隊 editor

**步驟**：
1. 以 editor 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{自己的user_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Left team"}`

### 測試案例 4：唯一 admin 嘗試離開

**前置條件**：使用者為團隊唯一的 admin

**步驟**：
1. 以 admin 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{自己的user_id}`

**預期結果**：
- HTTP 400
- 回傳 `{"detail": "Cannot leave: you are the only admin. Delete the team or assign another admin first."}`

### 測試案例 5：admin 離開（有其他 admin）

**前置條件**：團隊有多個 admin

**步驟**：
1. 以其中一位 admin 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{自己的user_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Left team"}`

### 測試案例 6：admin 取消 pending 邀請

**前置條件**：使用者為團隊 admin，團隊有一筆 pending invitation

**步驟**：
1. 以 admin 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{invitation_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invitation cancelled"}`

### 測試案例 7：移除不存在的成員

**前置條件**：使用者為團隊 admin

**步驟**：
1. 以 admin 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{不存在的id}`

**預期結果**：
- HTTP 404
- 回傳 `{"detail": "Member not found"}`
