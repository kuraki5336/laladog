# 移除成員 API

## 版本資訊

| 項目     | 說明                                                    |
| -------- | ------------------------------------------------------- |
| API 路徑 | `DELETE /teams/{team_id}/members/{member_user_id}`      |
| 版本     | 1.0                                                    |
| 最後更新 | 2026-03-03                                             |

## 相關資料表

| 資料表       | 用途                           |
| ------------ | ------------------------------ |
| team_members | 驗證操作者權限、查找並刪除成員記錄 |

## API 總覽

從團隊中移除指定成員。僅團隊擁有者（owner）可執行此操作。根據 `team_id` 與 `member_user_id` 查找對應的 `team_members` 記錄並刪除。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |

### Path Parameters

| 名稱           | 型別   | 必填 | 說明                          |
| -------------- | ------ | ---- | ----------------------------- |
| team_id        | string | 是   | 團隊 UUID                    |
| member_user_id | string | 是   | 要移除的成員 user ID（UUID）  |

### Query Parameters

無。

### Request Body

無。

### Request 範例

```http
DELETE /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890/members/u002-user-uuid HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## Response 規格

### 成功回應（200 OK）

| 欄位    | 型別   | 說明               |
| ------- | ------ | ------------------ |
| message | string | 移除成功訊息       |

### Response 範例

```json
{
  "message": "Member removed"
}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. **權限驗證**：查詢 `team_members` 表，確認當前使用者在該團隊中的角色為 `owner`
   - 若非 owner → 回傳 403
3. **查找目標成員**：查詢 `team_members` 表，以 `team_id` 與 `member_user_id` 查找成員記錄
   - 若找不到 → 回傳 404
4. 刪除該 `TeamMember` 記錄
5. 執行 `commit()`
6. 回傳成功訊息

### 權限控管

- 僅 `role == "owner"` 的團隊成員可執行移除操作
- editor 和 viewer 角色無法移除成員

### 注意事項

- 目前實作未限制 owner 移除自己，若 owner 移除自己將導致團隊無擁有者
- 刪除 `TeamMember` 記錄為硬刪除，不可還原
- 不會連帶刪除該使用者在團隊內的其他相關資料（如 SharedCollection）

### 程式碼位置

- Router：`backend/routers/teams.py` → `remove_member()`
- Model：`backend/models/team.py` → `TeamMember`

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                              | 觸發條件                              |
| ----------- | ------------------------------------- | ------------------------------------- |
| 401         | Invalid or expired token              | JWT Token 無效或過期                  |
| 403         | Not authenticated                     | 未提供 Authorization header           |
| 403         | Only team owner can remove members    | 操作者不是該團隊的 owner              |
| 404         | Member not found                      | 指定的 member_user_id 不在該團隊中    |

## 測試案例

### 測試案例 1：owner 成功移除成員

**前置條件**：使用者為團隊 owner，目標成員存在於該團隊中

**步驟**：
1. 以 owner 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{member_user_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Member removed"}`
- 資料庫中 `team_members` 表對應記錄已被刪除

### 測試案例 2：非 owner 嘗試移除成員（editor 角色）

**前置條件**：使用者為團隊 editor（非 owner）

**步驟**：
1. 以 editor 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{member_user_id}`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only team owner can remove members"}`

### 測試案例 3：非 owner 嘗試移除成員（viewer 角色）

**前置條件**：使用者為團隊 viewer（非 owner）

**步驟**：
1. 以 viewer 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{member_user_id}`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only team owner can remove members"}`

### 測試案例 4：移除不存在的成員

**前置條件**：使用者為團隊 owner

**步驟**：
1. 以 owner 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{不存在的user_id}`

**預期結果**：
- HTTP 404
- 回傳 `{"detail": "Member not found"}`

### 測試案例 5：未認證請求

**前置條件**：無

**步驟**：
1. 不帶 Authorization header 發送 `DELETE /teams/{team_id}/members/{member_user_id}`

**預期結果**：
- HTTP 403
- 回傳錯誤訊息

### 測試案例 6：owner 嘗試移除自己

**前置條件**：使用者為團隊 owner

**步驟**：
1. 以 owner 的 JWT Token 發送 `DELETE /teams/{team_id}/members/{自己的user_id}`

**預期結果**：
- HTTP 200（目前實作未限制此情境）
- 回傳 `{"message": "Member removed"}`
- **注意**：此操作會導致團隊無擁有者，建議未來版本加入限制
