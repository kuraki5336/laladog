# 邀請成員 API

## 版本資訊

| 項目     | 說明                                    |
| -------- | --------------------------------------- |
| API 路徑 | `POST /teams/{team_id}/invite`          |
| 版本     | 2.0                                    |
| 最後更新 | 2026-03-04                             |
| 狀態     | 已實作                                 |

## 相關資料表

| 資料表              | 用途                                 |
| ------------------- | ------------------------------------ |
| team_members        | 驗證操作者權限、檢查是否已為成員、新增成員記錄 |
| users               | 透過 email 查找受邀使用者            |
| pending_invitations | 為未註冊使用者建立待確認邀請記錄     |

## API 總覽

邀請使用者加入團隊。僅團隊管理員（admin）可執行此操作。透過受邀者的 email 查找使用者：

- **已註冊使用者**：直接建立 `team_members` 記錄加入團隊
- **未註冊使用者**：建立 `pending_invitations` 記錄，待對方註冊後自動加入

可指派的角色包含 `viewer`、`editor`、`admin` 三種。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |
| Content-Type  | 是   | `application/json`            |

### Path Parameters

| 名稱    | 型別   | 必填 | 說明             |
| ------- | ------ | ---- | ---------------- |
| team_id | string | 是   | 團隊 UUID        |

### Query Parameters

無。

### Request Body

| 欄位  | 型別   | 必填 | 說明                                           | 預設值     |
| ----- | ------ | ---- | ---------------------------------------------- | ---------- |
| email | string | 是   | 受邀使用者的 email                             | -          |
| role  | string | 否   | 指派角色：`viewer`、`editor` 或 `admin`        | `"viewer"` |

### Request 範例

```http
POST /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890/invite HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
  "email": "member@example.com",
  "role": "editor"
}
```

## Response 規格

### 成功回應（200 OK）— 已註冊使用者

| 欄位    | 型別   | 說明               |
| ------- | ------ | ------------------ |
| message | string | 邀請成功訊息       |

### Response 範例（已註冊）

```json
{
  "message": "Invited member@example.com as editor"
}
```

### Response 範例（未註冊）

```json
{
  "message": "Invitation sent to newuser@example.com (pending registration)"
}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. **角色驗證**：檢查 `body.role` 是否為 `viewer`、`editor`、`admin` 之一
   - 若不合法 → 回傳 400
3. **權限驗證**：查詢 `team_members` 表，確認當前使用者在該團隊中的角色為 `admin`
   - 若非 admin → 回傳 403
4. **查找受邀者**：查詢 `users` 表，以 `email` 欄位查找使用者
5. **分支處理**：
   - **已註冊**：
     a. 查詢 `team_members` 表，確認受邀者是否已為該團隊成員
     b. 若已為成員 → 回傳 409
     c. 建立 `TeamMember` 物件並寫入
   - **未註冊**：
     a. 查詢 `pending_invitations` 表，確認是否已有待確認邀請
     b. 若已有邀請 → 回傳 409
     c. 建立 `PendingInvitation` 物件並寫入
6. 執行 `commit()` 並回傳成功訊息

### 權限控管

- 僅 `role == "admin"` 的團隊成員可執行邀請操作
- editor 和 viewer 角色無法邀請成員

### 角色指派規則

- 可指派的角色為 `viewer`、`editor`、`admin`
- 預設角色為 `viewer`
- 角色值不合法時回傳 400 錯誤

### 程式碼位置

- Router：`backend/routers/teams.py` → `invite_member()`
- Model：`backend/models/team.py` → `TeamMember`, `PendingInvitation`
- Schema：`backend/routers/teams.py` → `InviteMemberRequest`
- 外部參照：`backend/models/user.py` → `User`（函式內 import）

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                             | 觸發條件                              |
| ----------- | ------------------------------------ | ------------------------------------- |
| 400         | Role must be viewer, editor, or admin | 角色值不合法                          |
| 401         | Invalid or expired token             | JWT Token 無效或過期                  |
| 403         | Not authenticated                    | 未提供 Authorization header           |
| 403         | Only admin can invite members        | 操作者不是該團隊的 admin              |
| 409         | User already in team                 | 受邀者已經是該團隊的成員              |
| 409         | Invitation already sent              | 已對該 email 發送過待確認邀請         |
| 422         | Validation Error                     | Request body 格式錯誤或缺少必填欄位   |

## 測試案例

### 測試案例 1：admin 成功邀請已註冊成員（預設角色）

**前置條件**：使用者為團隊 admin，受邀者已註冊且尚未加入該團隊

**步驟**：
1. 以 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "newmember@example.com"}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invited newmember@example.com as viewer"}`
- 資料庫中 `team_members` 表新增一筆 `role="viewer"` 的記錄

### 測試案例 2：admin 邀請已註冊成員並指定 editor 角色

**前置條件**：使用者為團隊 admin，受邀者已註冊且尚未加入該團隊

**步驟**：
1. 以 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "editor@example.com", "role": "editor"}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invited editor@example.com as editor"}`
- 資料庫中新增 `role="editor"` 的成員記錄

### 測試案例 3：admin 邀請未註冊使用者

**前置條件**：使用者為團隊 admin，受邀者尚未在系統中註冊

**步驟**：
1. 以 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "newuser@example.com", "role": "editor"}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invitation sent to newuser@example.com (pending registration)"}`
- 資料庫中 `pending_invitations` 表新增一筆記錄

### 測試案例 4：非 admin 嘗試邀請

**前置條件**：使用者為團隊 editor 或 viewer

**步驟**：
1. 以非 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only admin can invite members"}`

### 測試案例 5：邀請已為成員的使用者

**前置條件**：使用者為團隊 admin，受邀者已在該團隊中

**步驟**：
1. 以 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為已為成員的使用者 email

**預期結果**：
- HTTP 409
- 回傳 `{"detail": "User already in team"}`

### 測試案例 6：不合法的角色值

**前置條件**：使用者為團隊 admin

**步驟**：
1. 以 admin 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "user@example.com", "role": "superadmin"}`

**預期結果**：
- HTTP 400
- 回傳 `{"detail": "Role must be viewer, editor, or admin"}`
