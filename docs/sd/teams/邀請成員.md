# 邀請成員 API

## 版本資訊

| 項目     | 說明                                    |
| -------- | --------------------------------------- |
| API 路徑 | `POST /teams/{team_id}/invite`          |
| 版本     | 1.0                                    |
| 最後更新 | 2026-03-03                             |
| 狀態     | 已實作                                 |

## 相關資料表

| 資料表       | 用途                                 |
| ------------ | ------------------------------------ |
| team_members | 驗證操作者權限、檢查是否已為成員、新增成員記錄 |
| users        | 透過 email 查找受邀使用者            |

## API 總覽

邀請使用者加入團隊。僅團隊擁有者（owner）可執行此操作。透過受邀者的 email 查找已註冊的使用者，並建立一筆 `team_members` 記錄。若受邀者尚未註冊、或已為團隊成員，將回傳對應的錯誤。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |
| Content-Type  | 是   | `application/json`            |

### Path Parameters

| 名稱    | 型別   | 必填 | 說明             |
| ------- | ------ | ---- | ---------------- |
| team_id | string | 是   | 團隊 UUID        |

### Query Parameters

無。

### Request Body

| 欄位  | 型別   | 必填 | 說明                             | 預設值     |
| ----- | ------ | ---- | -------------------------------- | ---------- |
| email | string | 是   | 受邀使用者的 email               | -          |
| role  | string | 否   | 指派角色：`editor` 或 `viewer`   | `"viewer"` |

### Request 範例

```http
POST /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890/invite HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
  "email": "member@example.com",
  "role": "editor"
}
```

## Response 規格

### 成功回應（200 OK）

| 欄位    | 型別   | 說明               |
| ------- | ------ | ------------------ |
| message | string | 邀請成功訊息       |

### Response 範例

```json
{
  "message": "Invited member@example.com as editor"
}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. **權限驗證**：查詢 `team_members` 表，確認當前使用者在該團隊中的角色為 `owner`
   - 若非 owner → 回傳 403
3. **查找受邀者**：查詢 `users` 表，以 `email` 欄位查找使用者
   - 若找不到 → 回傳 404
4. **重複檢查**：查詢 `team_members` 表，確認受邀者是否已為該團隊成員
   - 若已為成員 → 回傳 409
5. 建立 `TeamMember` 物件，設定 `team_id`、`user_id`（受邀者）、`role`
6. 寫入資料庫並 `commit()`
7. 回傳成功訊息

### 權限控管

- 僅 `role == "owner"` 的團隊成員可執行邀請操作
- editor 和 viewer 角色無法邀請成員

### 角色指派規則

- 可指派的角色為 `editor` 或 `viewer`
- 預設角色為 `viewer`
- 不可透過邀請指派 `owner` 角色（目前實作未做此驗證，但語意上不應使用）

### 程式碼位置

- Router：`backend/routers/teams.py` → `invite_member()`
- Model：`backend/models/team.py` → `TeamMember`
- Schema：`backend/routers/teams.py` → `InviteMemberRequest`
- 外部參照：`backend/models/user.py` → `User`（函式內 import）

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                             | 觸發條件                              |
| ----------- | ------------------------------------ | ------------------------------------- |
| 401         | Invalid or expired token             | JWT Token 無效或過期                  |
| 403         | Not authenticated                    | 未提供 Authorization header           |
| 403         | Only team owner can invite members   | 操作者不是該團隊的 owner              |
| 404         | User not found                       | 以 email 查無已註冊使用者             |
| 409         | User already in team                 | 受邀者已經是該團隊的成員              |
| 422         | Validation Error                     | Request body 格式錯誤或缺少必填欄位   |

## 測試案例

### 測試案例 1：owner 成功邀請新成員（預設角色）

**前置條件**：使用者為團隊 owner，受邀者已註冊且尚未加入該團隊

**步驟**：
1. 以 owner 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "newmember@example.com"}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invited newmember@example.com as viewer"}`
- 資料庫中 `team_members` 表新增一筆 `role="viewer"` 的記錄

### 測試案例 2：owner 邀請成員並指定 editor 角色

**前置條件**：使用者為團隊 owner，受邀者已註冊且尚未加入該團隊

**步驟**：
1. 以 owner 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "editor@example.com", "role": "editor"}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Invited editor@example.com as editor"}`
- 資料庫中新增 `role="editor"` 的成員記錄

### 測試案例 3：非 owner 嘗試邀請（editor 角色）

**前置條件**：使用者為團隊 editor（非 owner）

**步驟**：
1. 以 editor 的 JWT Token 發送 `POST /teams/{team_id}/invite`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only team owner can invite members"}`

### 測試案例 4：非 owner 嘗試邀請（viewer 角色）

**前置條件**：使用者為團隊 viewer（非 owner）

**步驟**：
1. 以 viewer 的 JWT Token 發送 `POST /teams/{team_id}/invite`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only team owner can invite members"}`

### 測試案例 5：邀請未註冊的 email

**前置條件**：使用者為團隊 owner

**步驟**：
1. 以 owner 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為 `{"email": "nonexistent@example.com"}`

**預期結果**：
- HTTP 404
- 回傳 `{"detail": "User not found"}`

### 測試案例 6：邀請已為成員的使用者

**前置條件**：使用者為團隊 owner，受邀者已在該團隊中

**步驟**：
1. 以 owner 的 JWT Token 發送 `POST /teams/{team_id}/invite`
2. body 為已為成員的使用者 email

**預期結果**：
- HTTP 409
- 回傳 `{"detail": "User already in team"}`

### 測試案例 7：未認證請求

**前置條件**：無

**步驟**：
1. 不帶 Authorization header 發送 `POST /teams/{team_id}/invite`

**預期結果**：
- HTTP 403
- 回傳錯誤訊息
