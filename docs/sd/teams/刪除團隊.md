| 版本     | 1.0                                    |

## 相關資料表

| 資料表              | 用途                                         |
| ------------------- | -------------------------------------------- |
| teams               | 查找並刪除團隊記錄                           |
| team_members        | CASCADE 自動刪除（FK → teams.id）            |
| pending_invitations | CASCADE 自動刪除（FK → teams.id）            |
| shared_collections  | CASCADE 自動刪除（FK → teams.id）            |

## API 總覽

刪除整個團隊。僅團隊管理員（admin）可執行此操作。刪除時透過資料庫的 CASCADE 約束，自動刪除所有關聯的 `team_members`、`pending_invitations`、`shared_collections` 記錄。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |

### Path Parameters

| 名稱    | 型別   | 必填 | 說明             |
| ------- | ------ | ---- | ---------------- |
| team_id | string | 是   | 團隊 UUID        |

### Query Parameters

無。

### Request Body

無。

### Request 範例

```http
DELETE /teams/a1b2c3d4-e5f6-7890-abcd-ef1234567890 HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## Response 規格

### 成功回應（200 OK）

| 欄位    | 型別   | 說明               |
| ------- | ------ | ------------------ |
| message | string | 刪除成功訊息       |

### Response 範例

```json
{
  "message": "Team deleted"
}
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. **權限驗證**：查詢 `team_members` 表，確認當前使用者在該團隊中的角色為 `admin`
   - 若非 admin → 回傳 403
3. **查找團隊**：查詢 `teams` 表以 `team_id` 查找團隊記錄
   - 若找不到 → 回傳 404
4. 刪除 `Team` 記錄
5. 資料庫 CASCADE 自動刪除所有關聯記錄：
   - `team_members`（該團隊的所有成員）
   - `pending_invitations`（該團隊的所有待確認邀請）
   - `shared_collections`（該團隊的所有共享 Collection）
6. 執行 `commit()` 並回傳成功訊息

### 權限控管

- 僅 `role == "admin"` 的團隊成員可執行刪除操作
- editor 和 viewer 角色無法刪除團隊

### 注意事項

- 此操作不可還原，團隊及所有關聯資料將永久刪除
- 前端在呼叫此 API 後需同時移除本地工作區並中斷 WebSocket 連線

### 程式碼位置

- Router：`backend/routers/teams.py` → `delete_team()`
- Model：`backend/models/team.py` → `Team`

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                           | 觸發條件                              |
| ----------- | ---------------------------------- | ------------------------------------- |
| 401         | Invalid or expired token           | JWT Token 無效或過期                  |
| 403         | Not authenticated                  | 未提供 Authorization header           |
| 403         | Only admin can delete team         | 操作者不是該團隊的 admin              |
| 404         | Team not found                     | 指定的 team_id 不存在                 |

## 測試案例

### 測試案例 1：admin 成功刪除團隊

**前置條件**：使用者為團隊 admin

**步驟**：
1. 以 admin 的 JWT Token 發送 `DELETE /teams/{team_id}`

**預期結果**：
- HTTP 200
- 回傳 `{"message": "Team deleted"}`
- `teams` 表中對應記錄已刪除
- `team_members` 表中該團隊的所有記錄已 CASCADE 刪除
- `pending_invitations` 表中該團隊的所有記錄已 CASCADE 刪除
- `shared_collections` 表中該團隊的所有記錄已 CASCADE 刪除

### 測試案例 2：非 admin 嘗試刪除團隊

**前置條件**：使用者為團隊 editor 或 viewer

**步驟**：
1. 以非 admin 的 JWT Token 發送 `DELETE /teams/{team_id}`

**預期結果**：
- HTTP 403
- 回傳 `{"detail": "Only admin can delete team"}`

### 測試案例 3：刪除不存在的團隊

**前置條件**：使用者已登入

**步驟**：
1. 以有效 JWT Token 發送 `DELETE /teams/{不存在的team_id}`

**預期結果**：
- HTTP 403 或 404（取決於是否為任何團隊的 admin）

### 測試案例 4：未認證請求

**前置條件**：無

**步驟**：
1. 不帶 Authorization header 發送 `DELETE /teams/{team_id}`

**預期結果**：
- HTTP 403
- 回傳錯誤訊息
