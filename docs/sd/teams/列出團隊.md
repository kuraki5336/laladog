| 版本     | 1.0                            |

## 相關資料表

| 資料表       | 用途                         |
| ------------ | ---------------------------- |
| teams        | 查詢團隊基本資料             |
| team_members | 查詢使用者所屬團隊及其角色   |

## API 總覽

列出當前登入使用者所屬的所有團隊，包含使用者在各團隊中的角色。透過 `team_members` 表 JOIN `teams` 表，篩選出 `user_id` 等於當前使用者的所有記錄。

## Request 規格

### Headers

| 名稱          | 必填 | 說明                          |
| ------------- | ---- | ----------------------------- |
| Authorization | 是   | `Bearer {JWT Token}`          |

### Path Parameters

無。

### Query Parameters

無。

### Request Body

無。

### Request 範例

```http
GET /teams/ HTTP/1.1
Host: api.laladog.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## Response 規格

### 成功回應（200 OK）

回傳 JSON 陣列，每個元素為一個 `TeamResponse` 物件。

| 欄位     | 型別   | 說明                                |
| -------- | ------ | ----------------------------------- |
| id       | string | 團隊 UUID                          |
| name     | string | 團隊名稱                           |
| owner_id | string | 團隊擁有者的 user ID               |
| role     | string | 當前使用者在此團隊的角色（owner/editor/viewer） |
| members  | array  | 成員列表（預設為空陣列）            |

### Response 範例

```json
[
  {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "name": "研發一組",
    "owner_id": "u001-user-uuid",
    "role": "owner",
    "members": []
  },
  {
    "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "name": "設計團隊",
    "owner_id": "u002-user-uuid",
    "role": "viewer",
    "members": []
  }
]
```

### 空結果回應（200 OK）

使用者尚未加入任何團隊時，回傳空陣列。

```json
[]
```

## 實作說明

### 處理流程

1. 從 JWT Token 解析出當前使用者的 `sub`（即 `user_id`）
2. 查詢 `team_members` 表，JOIN `teams` 表，條件為 `TeamMember.user_id == user_id`
3. 將查詢結果逐一組裝為 `TeamResponse` 物件
4. 回傳團隊列表

### SQL 查詢邏輯

```sql
SELECT team_members.*, teams.*
FROM team_members
JOIN teams ON team_members.team_id = teams.id
WHERE team_members.user_id = :user_id
```

### 程式碼位置

- Router：`backend/routers/teams.py` → `list_teams()`
- Model：`backend/models/team.py` → `Team`, `TeamMember`

## 錯誤代碼

| HTTP 狀態碼 | 錯誤訊息                    | 觸發條件              |
| ----------- | --------------------------- | --------------------- |
| 401         | Invalid or expired token    | JWT Token 無效或過期  |
| 403         | Not authenticated           | 未提供 Authorization header |

## 測試案例

### 測試案例 1：成功列出使用者所屬團隊

**前置條件**：使用者已登入，且為兩個團隊的成員

**步驟**：
1. 以有效 JWT Token 發送 `GET /teams/`

**預期結果**：
- HTTP 200
- 回傳包含 2 個團隊的 JSON 陣列
- 每個團隊包含正確的 `role` 欄位

### 測試案例 2：使用者尚未加入任何團隊

**前置條件**：使用者已登入，但未加入任何團隊

**步驟**：
1. 以有效 JWT Token 發送 `GET /teams/`

**預期結果**：
- HTTP 200
- 回傳空陣列 `[]`

### 測試案例 3：未認證請求

**前置條件**：無

**步驟**：
1. 不帶 Authorization header 發送 `GET /teams/`

**預期結果**：
- HTTP 403
- 回傳錯誤訊息

### 測試案例 4：Token 過期

**前置條件**：使用已過期的 JWT Token

**步驟**：
1. 以過期 JWT Token 發送 `GET /teams/`

**預期結果**：
- HTTP 401
- 回傳 `{"detail": "Invalid or expired token"}`
