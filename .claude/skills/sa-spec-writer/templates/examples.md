# 欄位規格撰寫範例集

本文件收錄各種常見欄位類型的規格撰寫範例，供撰寫時參考。

---

## 畫面總覽範例

### 範例 A：證照定義編輯畫面

```markdown
## 畫面總覽

| 項目 | 說明 |
|-----|------|
| 畫面用途 | 供 HR 管理員新增或編輯證照定義資料，建立公司認可的證照清單 |
| 進入條件 | 從「證照定義列表」頁面，點擊「新增」按鈕或點擊某筆資料的「編輯」按鈕 |
| 操作權限 | 系統管理員、HR 主管 |
| 相關畫面 | 證照定義列表、員工證照管理 |
```

### 範例 B：Banner 設定畫面

```markdown
## 畫面總覽

| 項目 | 說明 |
|-----|------|
| 畫面用途 | 供行銷人員設定首頁 Banner 輪播內容，包含圖片或影片 |
| 進入條件 | 從「內容管理 > Banner 管理」，點擊「新增 Banner」或「編輯」 |
| 操作權限 | 行銷人員、內容管理員、系統管理員 |
| 相關畫面 | Banner 列表、首頁預覽 |
```

## 區域權限與用途範例

### 範例 C：標題區域

```markdown
### 標題區域

> **用途**：顯示當前操作模式（新增或編輯），讓使用者清楚知道正在進行的動作
> **權限**：所有可進入此畫面的角色皆可檢視
```

### 範例 D：基本資訊區域

```markdown
### 基本資訊區域

> **用途**：填寫項目的核心識別資料，這些欄位決定項目的基本屬性
> **權限**：具有編輯權限的角色可填寫；唯讀角色僅能檢視，欄位顯示為禁用狀態
```

### 範例 E：進階設定區域

```markdown
### 進階設定區域

> **用途**：提供進階參數調整，影響系統行為，一般使用者不需變更
> **權限**：僅系統管理員可操作；其他角色此區域隱藏或顯示唯讀
```

### 範例 F：操作按鈕區域

```markdown
### 操作按鈕區域

> **用途**：提供使用者儲存變更或取消操作
> **權限**：具有編輯權限的角色可操作；唯讀角色此區域隱藏
```

## 欄位類型範例

### 範例 1：純文字（多情境值）

**彈窗標題與副標題 欄位**

> *位於彈窗最頂部，依據操作情境動態變更文字內容*

* **數據類型：** 純文字
* **語言切換：** 無
* **必填：** NA
* **檢核：** 無
* **佔位符：** NA
* **值：**
  * **新增外部證照**
    * 標題：`新增外部證照`
    * 副標題：`請填寫外部證照的相關資訊`
  * **編輯外部證照**
    * 標題：`編輯外部證照`
    * 副標題：`請填寫外部證照的相關資訊`
  * **新增內部證照**
    * 標題：`新增內部證照`
    * 副標題：`請填寫內部證照的相關資訊`
  * **編輯內部證照**
    * 標題：`編輯內部證照`
    * 副標題：`請填寫內部證照的相關資訊`

## 範例 12：單選按鈕控制多區塊（互斥選項）

**Banner 圖片或影片 欄位**

> *圖片與影片只能擇一，上方出現單選按鈕提供選擇*

* **數據類型：** 單選按鈕
* **語言切換：** NA
* **必填：** 是
* **檢核：** 至少選其中一個
* **佔位符：** NA
* **值：** 影片 (預設)；圖片

**圖片 欄位**

* **備註：** 無

**影片代碼 欄位**

* **必填：** 是
* **檢核：** 中文上限為 `200` 個字；英文上限為 `200` 個字。
* **佔位符：** 中文顯示 `請輸入影片代碼`；英文顯示 `請輸入影片代碼`。
* **值：**
* **備註：**
  * 輸入框旁有問號提示圖示，若點擊將彈出影片代碼提示圖片
  * 文字輸入框離開焦點時出現預覽影片區塊，若預覽錯誤則無法播放或顯示錯誤

## 範例 20：列表 (Table)

**課程清單 列表**

> *顯示課程資料表格*

* **數據類型：** 列表 (Table)
* **語言切換：** 無
* **必填：** NA
* **檢核：** 無
* **佔位符：**
  * 無資料狀態：當系統初始無資料，或搜尋條件查無符合的課程時，列表區域顯示空狀態圖示 (Empty State)，並搭配文字 `暫無符合的課程` 、 `請重新調整搜尋條件。` 。
* **值：**
  * `課程名稱/代碼` ({別名名稱} + \n + {代碼})
  * `技能類別` ({技能別名})
  * `上課日期` (YYYY-MM-DD)
  * `班次狀態` ({狀態標籤})
  * `正取 / 備取人數` ({正取數} / {備取數})
  * `操作` (檢視)
* **備註：**
  * **預設排序：** 依照 **上課時間** 由新到舊 (Descending) 排序。
  * **資料來源**：課程基本資料由外部系統串接。
  * **勾選功能**：列表最左側包含 Checkbox，支援多選與單頁全選。
  * 列表欄位說明：
    1. **班次狀態 (依原始碼判斷)**：
       a. `未開始`：原始碼 0, 4, 5, 6, 9。
       b. `報名中`：原始碼 7。
       c. `報名截止`：原始碼 2, 10。
       d. `上課中`：原始碼 1。
       e. `已結訓`：原始碼 3, 8, 11。
    2. **正取 / 備取人數**：顯示目前的報名錄取狀況。
       a. 正取：指目前 **已成功錄取** 的實際學員人數。
       b. 備取：指目前在 **備取名單** 中等待遞補的學員人數。
    3. **操作**：
       a. `檢視`（藍色文字連結）：點擊後導向課程詳情頁面。

## 範例 21：列表含 Toggle 開關欄位

**選單列表 列表**

> *顯示選單項目資料表格，支援樹狀結構展開收合*

* **數據類型：** 列表 (Table)
* **語言切換：** 無
* **必填：** NA
* **檢核：** 無
* **佔位符：**
  * 無資料狀態：列表區域顯示空狀態圖示 (Empty State)，並搭配文字 `暫無選單資料`。
* **值：**
  * `（拖曳把手）` 六點拖曳圖示
  * `選單名稱` ({menuName})
  * `選單類型` ({menuTypeName})
  * `權限識別` ({permissionKey})
  * `路由網址` ({menuPath})
  * `頁面緩存` (Toggle 開關)
  * `顯示` (Toggle 開關)
  * `操作` (刪除、編輯)
* **備註：**
  * **資料來源**：由 API `GET /api/menus` 取得列表資料。
  * **預設排序**：按選單類型排序（目錄 → 選單），同類型按 `sortOrder` 升序排列。
  * 列表欄位說明：
    1. **頁面緩存** (Toggle 開關)：
       a. 紅色 = 啟用 (`keepAlive = true`)、灰色 = 停用 (`keepAlive = false`)。
       b. 點擊後立即呼叫 `POST /api/menus/:menuId/settings` 更新。
       c. 若 API 失敗，Toggle 恢復至原始狀態並顯示錯誤提示。
    2. **顯示** (Toggle 開關)：
       a. 紅色 = 顯示 (`showInMenu = true`)、灰色 = 隱藏 (`showInMenu = false`)。
       b. 點擊後立即呼叫 `POST /api/menus/:menuId/settings` 更新。
       c. 若 API 失敗，Toggle 恢復至原始狀態並顯示錯誤提示。
    3. **操作 - 刪除**（紅色文字連結）：
       a. 點擊後彈出刪除確認對話框：「確定要刪除此選單項目嗎？」
       b. 確認後呼叫 `DELETE /api/menus/:menuId`（標記刪除 `flag='D'`）。
       c. 若該選單下有子項目，顯示「該選單下有子項目，無法刪除」。
       d. 刪除成功後重新載入選單列表。
       e. 需具備 `menu:delete` 權限才顯示此按鈕。
    4. **操作 - 編輯**（藍色文字連結）：
       a. 點擊後呼叫 `GET /api/menus/:menuId` 取得選單詳情。
       b. 開啟「選單資訊」彈窗並預填資料。
       c. 編輯模式下，選單類型 Radio Button 設為 disabled，不可變更。
       d. 需具備 `menu:edit` 權限才顯示此按鈕。
